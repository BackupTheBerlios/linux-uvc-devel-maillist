> To keep the mailing list informed...

> 'uvcvideo' works now on my Gentoo Linux box. There were 2 issues:
> 1. Gentoo includes a patch for iSight, which breaks the driver (this
> is mentioned in the July mailing list). There is a bug report for this
> so hopefully they will make this patch optional.
> Is it possible to build-in proper support for the Apple iSight, rather
> than have a half working patch?

I remember Laurent was doing some tests with that patch but I'm not sure  
what the results were. I think part of it went into the change applied in  
r42 but I don't know if that's enough to make it work with the iSight.

You should know that the iSight doesn't even claim UVC support and from  
what I've heard the streaming headers are completely different. So  
distributions applying patches to the UVC driver for that is a rather bad  
idea unless properly done.

> 2. Taking the svn sources and apply the 'usb-reset' patch by Alexandre  
> Becoulet
> (https://lists.berlios.de/pipermail/linux-uvc-devel/2006-February/000210.html)
> This patch is a life saver. Without it the USB subsystem can become  
> unstable.
> Is it possible to include this patch into the main codebase, or
> otherwise add them to the berilios project page under section patches?

We're still trying to find a better (from a technical point of view)  
solution than what the USB reset patch provides. The error seems to be in  
the camera and we want to find a way to avoid triggering the USB errors.

In the meantime, adding the patch to the project page sounds like a good  
idea. I'll talk to Laurent when he returns (he's away for a couple of  
days).
