On Wed, 16 Aug 2006 15:27:31 -0700, Philip Heron <phil at firestorm.cx> wrote:

> On Tue, 2006-08-15 at 11:33 -0700, Martin Rubli wrote:
>> The control is a little special in that it's a bitmap (only a single bit
>> can be set at a time). You can see this in the spec (section 4.2.2.1.2):
>> 	http://www.usb.org/developers/devclass_docs/USB_Video_Class_1_1.zip
>>
>> Your camera should support Manual mode (bit 0) and Aperture Priority  
>> mode
>> (bit 3), so that 1 and 8 are the only valid values.
>
> Ah your right, I can set it to any of the first four bits and it sticks.
> I can't see any effect of setting it though.

Sorry for the delay, I was rather busy. But I did a little testing and the  
easiest way to spot the difference is to switch the value between 0 and 8  
while holding (possibly not perfectly still) the camera in a way that  
there's a bright light source in your background. You should see the  
backlight compensation kick in, i.e. the contrast of the foreground object  
should become better when set to 1 or 8.
