>> The feature is in the device itself so there's nothing special that the
>> driver has to do about it except supporting the corresponding UVC  
>> control.
>> Luckily, the uvcvideo driver does that with the help of the custom V4L2
>> control 'Exposure, auto' (V4L2_CID_EXPOSURE_AUTO). So if you have an
>> application that can toggle this control, it should work.
>
> Do all the uvcvideo supported Quickcams have this in the hardware? I'm
> only getting these controls from my Fusion:
>
> Available Controls        Current Value   Range
> ------------------        -------------   -----
> Brightness                127 (49%)       0 - 255
> Contrast                  32 (12%)        0 - 255
> Saturation                39 (15%)        0 - 255
> Gamma                     220 (100%)      100 - 220
> Gain                      16 (6%)         0 - 255
> Backlight Compensation    1               0 - 2
> Power Line Frequency      60 Hz           Disabled | 50 Hz | 60 Hz
> Sharpness                 180 (70%)       0 - 255
>
> I'd be interested in doing longer exposures if the camera could do it.

This is probably due to a bug in the way the enumeration of custom  
controls works (or doesn't work--it's at least partly a bug in the V4L2  
spec.) The attached patch should correct that, I've already submitted it  
to Laurent so it should be included soon.

BTW: The enumeration bug doesn't mean you can't use the EXPOSURE_AUTO  
control if the application hardcodes it ...

Can you try if you get more controls with this patch? If you're lucky  
you'll even get an absolute exposure time control but I'll have to double  
check when I'm back at the office.

Martin
