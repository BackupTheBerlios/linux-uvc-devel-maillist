Hi,

I am using Linux 2.6.17 compiled from the Debian package
linux-source-2.6.17 version 2.6.17-5. I compiled the Linux UVC driver
from SVN revision 44 (latest revision at this time).

BTW: I made my own Debian package which builds and runs on sarge
     (provided you're running a recent enough kernel) inspired from
     lirc-modules-source. It contains the Linux UVC SVN code, with no
     added patch. If people are interested, it's called linux-uvc-source
     and can be downloaded here:

       deb http://people.via.ecp.fr/~flo/debian sarge main
       deb-src http://people.via.ecp.fr/~flo/debian sarge main

I recently bought a Logitech Quickcam Pro 5000 and did my first tests
with ekiga. It worked well at first, but after a few minutes (I believe
it was after hanging up and calling right thereafter the sip:500 at
ekiga.net test account), the video stopped working and I found this in
my kernel logs:

  Aug  4 18:53:35 florent kernel: uvcvideo: Failed to query (1) UVC control 2 (unit 0) : -32.
  Aug  4 18:53:35 florent kernel: uvcvideo: Failed to query (1) UVC control 1 (unit 0) : -32.
  Aug  4 18:53:35 florent kernel: uvcvideo: Failed to query (1) UVC control 1 (unit 0) : -75.
  Aug  4 18:53:35 florent kernel: uvcvideo: Failed to query (1) UVC control 1 (unit 0) : -110.
  Aug  4 18:54:06 florent last message repeated 13 times
  Aug  4 18:54:40 florent last message repeated 5 times

Then, I rmmod'ed uvcvideo and got:

  Aug  4 18:58:31 florent kernel: usbcore: deregistering driver uvcvideo

When modprobing it after that, I got:

  Aug  4 18:58:39 florent kernel: uvcvideo: Found UVC 1.00 device <unnamed> (046d:08c5)
  Aug  4 18:58:39 florent kernel: uvcvideo: Failed to query (1) UVC control 1 (unit 0) : -110.
  Aug  4 18:58:39 florent kernel: uvcvideo: Failed to initialize the device (-5).
  Aug  4 18:58:39 florent kernel: usbcore: registered new driver uvcvideo
  Aug  4 18:58:39 florent kernel: USB Video Class driver (v0.1.0)

and of course, the webcam was still in a unusable state. This happens
everytime I do an rmmod/modprobe cycle (I didn't try to reboot yet).

Thanks for the driver, and for any help towards solving this issue.
