On Wed, May 24, 2006 at 06:23:11PM +0200, michel Xhaard wrote:

> > as I hobby, I'm doing robotics research. So far, we've used an Orange
> > Micro iBot2 webcam on our robot, which has an OV7620 sensor connected to
> > USB 2.0 via a simple bridge implemented in the Cypress Semi EzUSB FX2
> > microprocessor.
> >
> > This setup allows us to get the raw unprocessed data from the OV7620 VGA
> > CMOS sensor in Bayer format - one byte per pixel, when the sensor is
> > programmed right (via its i2c registers). (We also were able to do full
> > VGA resolution at 60 fps, but I don't expect that'll be possible on any
> > other camera ...)

> Just to be curious, what is the CPU/Hardware able to carry a VGA stream at 60 
> fp/s on your robot ? 

Oh, an Athlon64 at 2GHz was able to do that, and process the data as
well. 60fps at VGA resolution in Bayer format is some 18 megabytes of
data per second - it's easily feasible.

Note: Zero-copy delivery to userspace is a must at these data rates, so
my driver for the iBot delivers data directly via DMA to the processing
application, without ever copying it inside the kernel.

> > So the question is: Is it possible to get the raw data out of the
> > QuickCam Pro 5000 CCD VGA sensor?

> I did not know the QC Pro 5000 but the QC Pro for Notebook from
> Logitech can be set to provided 1024x576 in YUYV mode this webcam size
> is also a good point for a robot :)

Yes, but 1024x576 YUYV is still 35 megabytes per second at 30fps, twice
as much data for half the frame rate. I admit that the resolution is
much nicer than VGA, but that is not something I need that much.

My primary objectives are: Vivid colors (for good object recognition)
and high frame rate (for fast response of the robot).
